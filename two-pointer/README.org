#+title: Readme
* Two pointers:
These problems require two different pointers in order to calculate the final solution.
* Problems:
** [[https://leetcode.com/problems/middle-of-the-linked-list/description/][876. Middle of the Linked List]]

Assumptions: If the list length is odd the middle node is unique, if even we return the second middle node. No cycles.

Range & format:

The number of nodes in the list is in the range [1, 100]
1 <= Node.val <= 100

The format is a leetcode class:
#+BEGIN_SRC python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
#+END_SRC

On a high level the solution to this problem is to have two pointers. The first one will help us traverse the full list and calculate the lenght of the list. Then the second pointer will traverse until half the lenght and return the middle node.

Let's discuss some toy examples.


[0, 1] -> [1] i.e. index 1 = 2//2 of length 2

[0] -> [0] i.e. index 0 = 1//2 of length 1

[0,1,2] -> [1,2] i.e. index 1=3//2 of length 3

[0,1,2,3] -> [2,3] i.e. index 2=4//2 of length 4

#+BEGIN_SRC python
class Solution:
    def middleNode(self, head: Optional[ListNode]) -> Optional[ListNode]:
        first_pointer = head
        list_length = 0

        while first_pointer:
            list_length +=1
            first_pointer = first_pointer.next
        
        index = 0
        second_pointer = head
        while index < list_length//2:
            second_pointer = second_pointer.next
            index += 1

        return second_pointer
#+END_SRC
